## 📝 Template Checklist (For Each Problem)

Use this checklist when updating each problem:

- [ ] **Problem Statement**: Clear description with all details
- [ ] **Constraints**: All input/output constraints listed
- [ ] **Example 1**: Standard case with explanation
- [ ] **Example 2**: Different scenario with explanation
- [ ] **Example 3**: Edge case with explanation
- [ ] **Solution**: Well-commented code
- [ ] **Complexity**: Time and space analysis
- [ ] **Dry Run**: Step-by-step execution
- [ ] **Test Case 1**: Standard case
- [ ] **Test Case 2**: Edge case (empty/null)
- [ ] **Test Case 3**: Minimum size
- [ ] **Test Case 4**: Maximum size
- [ ] **Test Case 5**: Special values (negative/zero/duplicates)
- [ ] **Test Case 6**: Additional edge case
- [ ] **Use Cases**: 3-4 real-world applications
- [ ] **Key Insights**: Important observations
- [ ] **Common Mistakes**: What to avoid
- [ ] **Follow-ups**: Variations of problem
