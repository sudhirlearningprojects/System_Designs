## 💡 Amazon-Specific Tips

### 1. Leadership Principles Integration
Amazon expects you to demonstrate leadership principles while coding:
- **Customer Obsession**: Ask clarifying questions about use cases
- **Ownership**: Write clean, maintainable code
- **Invent and Simplify**: Start with brute force, then optimize
- **Dive Deep**: Explain time/space complexity thoroughly

### 2. Communication Style
- Think aloud while coding
- Explain your approach before coding
- Discuss trade-offs between solutions
- Ask for feedback during the interview

### 3. Code Quality
- Use meaningful variable names
- Add comments for complex logic
- Handle edge cases explicitly
- Write modular, testable code

### 4. Testing Approach
- Start with simple test cases
- Cover edge cases (empty, single element, large input)
- Walk through your code with test cases
- Identify and fix bugs proactively

## 📝 Interview Process Overview

### Amazon SDE Interview Rounds

**Round 1: Online Assessment (OA)**
- 2 coding problems (70 minutes)
- Focus on arrays, strings, and basic algorithms
- Must pass to proceed

**Round 2-4: Technical Phone/Video Screens**
- 1 coding problem per round (45 minutes)
- Live coding with interviewer
- Discuss approach, code, test, optimize

**Round 5-6: Onsite/Virtual Onsite**
- 4-5 rounds (45 minutes each)
- 2-3 coding rounds
- 1 system design round (for senior roles)
- 1 behavioral round (leadership principles)

### Typical Problem Breakdown
- **OA**: Easy to Medium (from this list: problems 1-15)
- **Phone Screen**: Medium (from this list: problems 10-25)
- **Onsite**: Medium to Hard (from this list: problems 15-30)
